# GCAM Project Rules

**Read before doing anything. These exist because violations have cost hours of compute.**

---

**1. NEVER EDIT BASE XML FILES.** Copy → modify copy → point config at copy. Only files Claude created are editable. The stock files are the ground truth for publication — never touch them.

**2. ANALYZE FIRST.** Read relevant files before any change. Glob before creating. Never assume.

**3. ONE CHANGE AT A TIME.** Explain it, wait for confirmation, then do it. No batching. "Let's do X" is NOT authorization — it is an invitation to discuss. Ask before creating any file, running any research agent, or taking any action beyond reading.

**4. NEVER MENTION THE NEXT RUN** until current run has: (1) exited cleanly, (2) name confirmed in DB, (3) no "did not solve correctly" in log.

**5. NEVER TOUCH THE DB WITHOUT EXPLICIT CONFIRMATION.** List all entries, map each timestamp to its run, state the plan explicitly, wait for "yes do it." A correctly-produced SSP1-2.6 run was deleted this way and was not recoverable.

**6. NEVER CLAIM SUCCESS WITHOUT VERIFYING.** Check the log. Check the DB. Not the code.

**7. GLOB BEFORE CREATING.** GCAM ships files for all standard scenarios. Check first.

**8. CLAUDE PRINTS COMMANDS. PARKER RUNS THEM.** Claude never directly executes: GCAM runs, kill/pkill, rm, or any DB modification. Print the command, stop. **Read-only checks (DB queries, log reads, file checks) Claude runs directly — do not ask Parker to run these.**

**9. RUN ATTEMPT LOG.** At the start of every session: read `memory/project_run_attempt_log.md` AND read `output/run_history.txt` (auto-generated by gcam_run.sh). Update the attempt log immediately after every run — success, failure, or Claude mistake. Use `bash gcam_run.sh <config> <label>` for all future runs so start/end/exit is auto-logged.

**10. THINK LIKE A SENIOR ENGINEER.** Before any action: read the relevant files, understand the full system, identify the minimal change needed. You are bad at this and have cost hours of compute through impulsive changes. When in doubt, do nothing and ask. The cost of pausing is zero. The cost of a wrong action is hours.

---

**Notebook style:** code and output only, no markdown cells. Reference in grey `#aaaaaa`, SSPs in color, no markers, clean axes.
